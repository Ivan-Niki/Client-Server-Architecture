------------------- FTP — что это такое? ------------------

FTP (File Transfer Protocol) — это протокол передачи файлов, который используется для передачи файлов между компьютерами в сети. 
FTP позволяет пользователям загружать файлы с удаленного компьютера или сервера на свой локальный компьютер или сервер, а также загружать файлы на удаленный сервер. 
Протокол FTP широко используется для управления веб-сайтами, обмена файлами и резервного копирования данных.

FTP работает на основе клиент-серверной архитектуры, где клиентское приложение, такое как FTP-клиент, 
устанавливает соединение с сервером, чтобы выполнить операции передачи файлов. 
FTP-клиент обеспечивает интерфейс для пользователя, который позволяет выполнить различные операции, такие как 
загрузка и скачивание файлов, удаление файлов, создание директорий и многое другое.

FTP использует порт 21 для установления соединения между клиентом и сервером. 
После установления соединения по порту 21, FTP-клиент и FTP-сервер могут использовать различные порты для передачи данных, в зависимости от конкретной операции. 
Например, для передачи файлов FTP может использовать порты 20 или 21, а для управления соединением может использоваться порт 21.
Протокол FTP поддерживается различными операционными системами и является одним из самых распространенных протоколов передачи файлов в сети интернет. 
FTP имеет множество функций и возможностей, которые делают его удобным инструментом для передачи файлов и управления удаленными серверами.

-------- Описание протокола передачи файлов --------
Протокол передачи файлов (FTP, File Transfer Protocol) является одним из самых распространенных протоколов для передачи файлов между клиентом и сервером через сеть TCP/IP. 
Этот протокол позволяет пользователям загружать файлы на сервер и скачивать их с сервера на свой компьютер.

FTP работает по клиент-серверной архитектуре, где клиентом является устройство, на котором запущен FTP-клиент, 
а сервером – устройство, на котором запущен FTP-сервер. 
Общение между клиентом и сервером осуществляется через предопределенные команды и ответы.

Протокол FTP использует два порта для передачи данных: порт управления (порт команд, по умолчанию 21) 
и порт данных (порт передачи данных, по умолчанию 20). 
Порт управления используется для обмена командами между клиентом и сервером, а порт данных – для фактической передачи самих файлов.

----- Основные команды, поддерживаемые протоколом FTP, включают: -----
USER: используется для указания имени пользователя при подключении к FTP-серверу.
PASS: применяется для указания пароля пользователя при подключении к FTP-серверу.
LIST: позволяет получить список файлов и папок на FTP-сервере.
RETR: осуществляет загрузку файла с FTP-сервера на клиентскую машину.
STOR: применяется для загрузки файла с клиентской машины на FTP-сервер.
DELE: удаляет файл с FTP-сервера.
MKD: создает новую папку на FTP-сервере.


----- Недостатки FTP -----
Самым большим недостатком этого протокола является незащищенность канала передачи данных. 
В то время, когда протокол создавался, считалось, что перехватить передаваемые по нему данные технически 
невозможно, поэтому встроенная защита в нем отсутствует. 
Иными словами, сегодня по FTP все данные передаются открыто, в том числе файлы, логины и пароли. 
Очевидно, что в современных условиях это абсолютно небезопасно: любой сторонний пользователь может подключиться 
к Wi-Fi-трафику или локальной сети и получить свободный доступ к информации.

Несмотря на то, что для доступа к данным в FTP нужно предоставить серверу логин и пароль, сама система авторизации остается незащищенной. 
То есть злоумышленник при желании может просто подобрать распространенные логины и пароли, чтобы получить доступ к данным. 
К тому же сейчас такой подбор делается не медленно и вручную, а с помощью генераторов ключей, которые анализируют за секунды тысячи возможных комбинаций.
